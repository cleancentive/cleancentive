# Feedback Plans

Local implementation plans generated by `/triagato` from production feedback.

## How it works

1. Run `/triagato` in Claude Code or OpenCode
2. The command fetches open feedback from PROD, compares against existing plans, and creates new ones
3. Review the generated plans in this directory
4. To implement a plan, tell your agent: "Implement the plan in docs/feedback-plans/{id}.md"
5. After implementation, update the plan's `status` to `implemented`

## Plan file format

Each `.md` file uses YAML frontmatter:

```yaml
---
feedback_id: <uuid>          # Links to PROD feedback item
title: <summary>              # Agent-generated short title
status: open | planned | implemented | wont-fix
category: bug | suggestion | question
severity: critical | high | medium | low
prod_status: new | acknowledged | in_progress | resolved
created: <ISO date>
last_synced: <ISO date>       # Last time triagato updated this from PROD
last_updated: <ISO date>      # Last time plan content changed
related: []                   # UUIDs of related feedback plans
---
```

## Authentication

triagato uses a device code flow to authenticate with PROD. On first run, it generates a 6-digit code and opens a browser page where you approve the login. The session token is saved to `infrastructure/.feedback-token` and reused across runs (365-day expiry).

## Notes

- Plan files (except this README) are **gitignored** — they're local working documents
- triagato is **read-only** against PROD — it never modifies feedback status
- Files are named `{feedback-id}.md` for easy cross-referencing
