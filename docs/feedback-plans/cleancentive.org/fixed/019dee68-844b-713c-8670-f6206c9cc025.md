## Analysis

The reporter clarified during planning that descriptions are typically copied from the public CleanCentive Google Forms ([form 1](https://docs.google.com/forms/d/e/1FAIpQLSepGAHHt1iybDBN7Rt4zXjRcavgxML6bfyOtmVcXya8ic_qzg/viewform), [form 2](https://docs.google.com/forms/d/e/1FAIpQLSf2ObioOP3bUHg_fqHrjMY7alhuyB4B_5JRx3eiFPdvkcw_EQ/viewform)). Pasting from those forms into the cleanup-description `<textarea>` strips HTML automatically (browser behaviour) and leaves plain text with embedded `\n` line breaks. The textarea preserves those newlines on save (verified in [backend/src/cleanup/cleanup.service.ts:154,170](../../../backend/src/cleanup/cleanup.service.ts) — only `trim()` is applied).

The bug is in the **read-only render**, not the textarea: [frontend/src/components/CleanupDetail.tsx:434](../../../frontend/src/components/CleanupDetail.tsx) renders `{cleanup.description && <p>{cleanup.description}</p>}` with no `white-space` style, so HTML's normal whitespace collapsing flattens every newline into a single mushed paragraph. That is what users see on the Adelboden cleanup detail page.

The codebase already uses `white-space: pre-wrap` for analogous text bodies — `.message-body` ([frontend/src/App.css:2444-2449](../../../frontend/src/App.css)) and `.feedback-detail-description` ([frontend/src/App.css:3609-3612](../../../frontend/src/App.css)). The fix reuses that convention.

Switching to a rich-text editor was considered (Tiptap/Lexical/Markdown) but rejected for this ticket — see the planning record for the brainstorm. Should be filed as a separate RFC together with the event-import wizard idea, referencing [docs/federation-analysis.md](../../../docs/federation-analysis.md) Phase 2.

## Implementation Plan

1. Add `className="cleanup-description-display"` to the `<p>` at [frontend/src/components/CleanupDetail.tsx:434](../../../frontend/src/components/CleanupDetail.tsx).
2. Append a CSS rule in [frontend/src/App.css](../../../frontend/src/App.css) near `.feedback-detail-description`:
   ```css
   .cleanup-description-display {
     white-space: pre-wrap;
     overflow-wrap: anywhere;
     margin: 0.5rem 0;
   }
   ```
   `overflow-wrap: anywhere` defends against pasted long unbroken URLs.
3. Typecheck `bun --cwd frontend tsc --noEmit`.
4. Manually verify on dev server: Adelboden cleanup renders multi-line; pasting a multi-line snippet from the Google Forms into a new cleanup round-trips with line breaks preserved.

No backend or schema changes. No textarea changes (creation/edit textareas are fine as-is).
