## Analysis
Two compounding bugs make the BaselArea favicon "stick" on the browser tab even after switching teams:

1. **No real default favicon ships.** [frontend/index.html:5](../../../frontend/index.html) references `/vite.svg` (Vite default — does not exist in the repo). [frontend/src/stores/partnerBrandingStore.ts:4](../../../frontend/src/stores/partnerBrandingStore.ts) sets `DEFAULT_FAVICON_HREF = '/favicon.ico'`, but no `/favicon.ico` exists in `frontend/public/` either.
2. **Reset to default silently 404s.** When a user on a partner team (e.g. BaselArea) switches to a different/no team, `applyPartnerBranding()` calls `updateFavicon('/favicon.ico')`. The browser fails to load the missing default and keeps showing the previously cached partner favicon — so the BaselArea logo persists.

The runtime override mechanism itself is desired and stays in place; only the missing default-favicon assets and stale references need fixing. The PWA manifest in [frontend/vite.config.ts:9-17](../../../frontend/vite.config.ts) also has no `icons` array, so installed PWA tiles lack a CleanCentive icon.

The CleanCentive logo SVG already exists at `visuals/cleancentive_logo.svg` and can be reused as the favicon source. Modern browsers and iOS Safari (16.4+) support SVG favicons; PNG/ICO fallbacks are intentionally skipped.

## Implementation Plan
1. **Add a CleanCentive favicon asset.**
   - Copy `visuals/cleancentive_logo.svg` to `frontend/public/favicon.svg`. Single asset only — no `.ico`, no PNG sizes, no apple-touch-icon.
2. **Fix the static favicon reference in `index.html`.**
   - [frontend/index.html:5](../../../frontend/index.html): change `href="/vite.svg"` → `href="/favicon.svg"`.
3. **Fix `DEFAULT_FAVICON_HREF` in the partner-branding store.**
   - [frontend/src/stores/partnerBrandingStore.ts:4](../../../frontend/src/stores/partnerBrandingStore.ts): change `'/favicon.ico'` → `'/favicon.svg'`. No other store changes — the existing subscribe → `applyPartnerBranding()` flow already resets to `DEFAULT_FAVICON_HREF` on team change / no-CSS / no-partner branches; the bug was purely the URL pointing to a missing file.
4. **Add PWA manifest icons.**
   - [frontend/vite.config.ts](../../../frontend/vite.config.ts): add an `icons` array referencing `/favicon.svg` (sizes `any`, type `image/svg+xml`, purpose `any maskable`).
