## Analysis
The user sees an empty map despite expecting spots. The curl command reveals the API is filtering by `cleanup_date_id=019d354e-ad76-7095-a97c-309f908d7e50`. 

Root cause: `MapPage.tsx:18-29` automatically applies `user.active_cleanup_date_id` as a filter. If the user's active cleanup has no spots associated with it, the map appears empty. The user says "no filters" but the cleanup context bar silently applies one.

This is a UX clarity issue — the active cleanup filter is not obviously communicated as a map filter. The user may not realize their cleanup selection is hiding spots.

Two possible fixes:
1. **Show a clear empty state** when the cleanup filter yields no results, explaining that the filter is active and how to clear it
2. **Don't auto-filter the map** by active cleanup — let the user opt in via a visible filter toggle

## Implementation Plan
1. In `MapPage.tsx`, add an empty state when `mapData` has 0 features and a `cleanup_date_id` filter is active — show "No spots for this cleanup. [Show all spots]" with a button to clear the filter
2. In `ContextBar.tsx`, make the active cleanup filter's effect on the map more visible (e.g., a pill/badge on the map tab)
3. Consider whether the map should default to showing all spots and let the cleanup filter be opt-in
