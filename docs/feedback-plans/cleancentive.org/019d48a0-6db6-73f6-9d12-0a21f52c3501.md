## Analysis
The map currently uses several overlapping color schemes in `MapPage.tsx`:
- **Cluster circles** (lines 86-90): gradient from `#51bbd6` → `#f1f075` → `#f28cb1` based on point count
- **Individual spots** (lines 125-134): blue for spots, green/orange/red for processing status
- **Cleanup locations** (line 154): fixed `#1976d2` blue

The user wants:
1. **Consistent color per entity type**: cleanups, spots, and pics should each have a distinct, recognizable color
2. **Count-based shading**: darker/more intense shades for higher counts (likely referring to clusters)

This is a visual design task. The current scheme mixes entity-type colors with processing-status colors, making it hard to distinguish what's what at a glance.

## Implementation Plan
1. Define a color palette with one base color per entity type:
   - Spots: blue family (e.g., `#2563eb`)
   - Cleanups: green family (e.g., `#16a34a`)
   - Clusters: use the entity type's base color with opacity/shade varying by count
2. Update `MapPage.tsx` layer styles:
   - Cluster circles: shade based on count using the spot base color
   - Individual spots: use spot base color (remove processing-status coloring or move to a secondary indicator)
   - Cleanup markers: use cleanup base color
3. Add a simple map legend showing the color → entity type mapping
